# Agent Working Rules

This repository is a long-running product build. Before making changes, always read:

1. `docs/DEVELOPMENT_PROGRESS.md`
2. `docs/PROJECT_PLAN.md`

At the end of every development session, update `docs/DEVELOPMENT_PROGRESS.md` with:

- Completed work
- Files changed
- Verification performed
- Current blockers
- The exact next tasks

Do not rely on chat history as the source of truth. The persistent docs are the source of truth.

Keep persistent planning and progress documents in ASCII English so they remain readable in PowerShell output. User-facing product copy should stay Chinese unless the user asks otherwise.
